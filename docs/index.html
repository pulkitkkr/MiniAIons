Capabilities

Everything you need to orchestrate agents.

Six tightly integrated surfaces, one local binary. No cloud dependency, no account, no usage cap beyond your own Anthropic plan.

Parallel Sessions

Spawn as many Claude conversations as you need. Each runs in its own tab, streams over WebSocket, and preserves full history to JSONL on disk.

WebSocket JSONL Hot swap

Projects

Group sessions by topic. Each project has its own color, system prompt, and session pool — drag-and-drop between projects to reorganize.

Custom Agents

Describe the role.
Claude writes the agent.

The Agent Builder is a conversation — not a form. Tell Claude what you want; it drafts a full agent definition with a pinned model, scoped toolset, and MCP bindings. Saved as a markdown file in ~/.claude/agents/ so the Claude CLI picks it up everywhere.

  • Pin any Claude model per agent — use Haiku for fast triage, Opus for synthesis
  • Scope exact tools — restrict a research agent to Read, Grep, WebSearch
  • Attach MCPs — wire in GitHub, Linear, Notion, Airbnb, or your own server
  • Ships with 7 starter agents: code-reviewer, debugger, architect, health-coach, research-assistant, shopping-advisor

Multi-step Workflows

Pipelines you can talk to at every step.

Workflows aren't automated black boxes. Each step opens a real Claude session you can steer in chat — when you're happy, send [STEP_COMPLETE] or click the complete button. Step output auto-pipes into the next prompt via template variables.

  • Variable interpolation: {{input}}, {{step_N_output}}
  • DAG dependencies — skip, parallel, fan-out patterns
  • Aggregated cost per run, broken down per step
  • 4 preset workflows: Trip Planner, Smart Purchase, Budget Check-Up, Wellness Plan
MCP Marketplace

Install plugins with one click.

Three marketplace tabs covering every shape of MCP server — remote HTTP/SSE, local NPM, and hand-rolled custom commands. All installs flow through the Claude CLI (claude mcp add), so what you install here is available everywhere else Claude runs.

  • Remote — hosted HTTP/SSE servers, zero config, instant connect
  • Local — NPM packages run via npx, sandboxed
  • Custom — your own binary with env vars, fields, and secrets

Configuration

Any Claude model. Any provider, eventually.

MiniAIons is provider-agnostic at its core. Models and CLIs are defined in config/providers.json — add a new entry and the UI picks it up automatically.

config/providers.json JSON 
{
  "claude": {
    "name": "Claude Code",
    "command": "claude",
    "args": ["-p", "--output-format", "stream-json"],
    "healthCheck": "claude --version",
    "models": [
      { "id": "opus",   "inputCostPer1M": 15 },
      { "id": "sonnet", "inputCostPer1M": 3 },
      { "id": "haiku",  "inputCostPer1M": 0.25 }
    ],
    "defaultModel": "sonnet"
  }
  // Add a new provider (e.g. copilot) here
  // and the Dashboard picks it up on reload.
}
